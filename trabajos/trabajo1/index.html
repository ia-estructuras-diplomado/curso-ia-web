    Trabajo Grupal 1: Análisis de Datos Estructurales¶

    📅 Evaluación 1

    Sesión: 4 | Fecha: Jueves, 14 de Agosto de 2026 | Peso: 30%

    🎯 Objetivo¶

    Aplicar conceptos fundamentales de Machine Learning para analizar datos reales de una estructura, identificando patrones y creando un modelo predictivo básico.

    📊 Contexto¶

    Tenéis datos de sensores de un puente o edificio con mediciones de: - Desplazamientos - Aceleraciones - Temperaturas - Estado clasificado manualmente

    Tu tarea: Construir un modelo que prediga el estado de la estructura basándose en los datos de sensores.

    📋 Componentes¶

    1. Presentación (15 minutos)¶

    • Introducción del problema (2 min)
    • Exploración de datos (3 min)
    • Modelo y métodos (5 min)
    • Resultados e insights (4 min)
    • Preguntas (1 min)

    2. Notebook Jupyter¶

    Debe incluir secciones: 1. Carga de datos 2. Exploración (EDA) 3. Preparación de datos 4. Entrenamiento de modelo 5. Evaluación 6. Conclusiones

    3. Reporte Escrito (2-3 páginas)¶

    Estructura: - Portada (1 página) - Resumen (párrafo) - Introducción (½ página) - Metodología (½ página) - Resultados (½ página) - Conclusiones (½ página) - Referencias

    📚 Recursos Disponibles¶

    • Dataset: estructura_001.csv (disponible en el repositorio)
    • Ejemplo Notebook: Ver Lab 1
    • Librerías: Pandas, NumPy, Scikit-learn

    ✅ Requisitos Técnicos¶

    • Cargar y explorar los datos
    • Calcular estadísticas descriptivas
    • Visualizar distribuciones (histogramas, boxplots)
    • Identificar correlaciones
    • Normalizar/preparar datos
    • Dividir en train/test (80/20)
    • Entrenar al menos 1 modelo (ej: Random Forest, Logistic Regression)
    • Evaluar con métricas (Accuracy, Precision, Recall, F1)
    • Crear matriz de confusión
    • Interpretar resultados

    📋 Checklist de Entrega¶

    Antes de las 23:59 del 14 de Agosto:

    • Presentación preparada (PPT/PDF)
    • Notebook funcionando y reproducible
    • Reporte escrito en PDF/Markdown
    • Todo subido al repositorio
    • Probé que todo funciona desde cero
    • Nombres de todos los miembros en documentos
    • Agradecimientos a fuentes/referencias

    🎓 Criterios de Evaluación¶

    Aspecto Excelente (4.5-5.0) Muy Bueno (4.0-4.4) Bueno (3.5-3.9) Satisfactorio (3.0-3.4) Insuficiente (<3.0)
    Comprensión Entienden completamente, explican bien Buena comprensión, explicaciones claras Comprensión sólida Comprensión básica No entienden
    Análisis Análisis profundo y riguroso Análisis adecuado Análisis presente Análisis superficial Sin análisis
    Técnica Implementación perfecta Implementación correcta Implementación con errores menores Algunos errores Errores graves
    Comunicación Excelente, muy clara Buena, clara Adecuada Poco clara Confusa
    Pensamiento Crítico Reflexión profunda Reflexión adecuada Algo de reflexión Poca reflexión Ninguna

    💡 Tips para Éxito¶

    1. Empezar temprano - No dejes todo para el último día
    2. Dividir el trabajo - Cada miembro en una parte
    3. Comunicación - Reúnete regularmente con el grupo
    4. Experimentar - Prueba diferentes modelos, no solo uno
    5. Documentar - Deja claro por qué haces cada paso
    6. Revisar - Chequea el código y reporte antes de entregar
    7. Ensayar - Practica la presentación varias veces

    🤔 Preguntas de Análisis¶

    Responde en tu reporte:

    1. ¿Qué sensor es más importante para predecir el estado?
    2. ¿El modelo comete errores sistemáticos?
    3. ¿Cómo mejorarías el modelo con más datos?
    4. ¿Es este modelo útil en la práctica? ¿Por qué?
    5. ¿Qué limitaciones tiene?
